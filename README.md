# 🥁 نظام إدارة فرقة المزيود الحربية

نظام متكامل لإدارة الأعضاء والمناسبات والصندوق المالي، مع بوابة خاصة للأعضاء لتلقي الدعوات والرد عليها بشكل فوري.

---

## ✨ المميزات

- 👥 إدارة كاملة للأعضاء والفئات والمناطق
- 📅 تسجيل المناسبات وحساب القسمة والمصاريف تلقائياً
- 💰 صندوق مالي مع تقارير مفصلة قابلة للطباعة
- 🔔 **بوابة العضو**: كل عضو يسجّل دخول ويرى دعواته ويردّ بـ "قبول" أو "اعتذار"
- ⚡ **مزامنة فورية بين كل الأجهزة** عبر Firebase (مجاني)
- 🎴 طباعة بطاقات عضوية أنيقة بحجم بطاقة الهوية القياسية
- 🔐 صلاحيات متعددة: مدير نظام، إداري، مدخل بيانات، عضو

---

## 🚀 طريقة التشغيل (3 خطوات)

### الطريقة السريعة (للاستخدام على جهاز واحد فقط)
1. حمّل المجلد كاملاً
2. افتح ملف `index.html` بأي متصفح حديث
3. سجّل الدخول بـ: `admin` / `admin`

> ⚠️ في هذه الطريقة، كل جهاز له بياناته المنفصلة. لمشاركة البيانات بين أجهزة، تابع للأسفل.

---

## 🌍 الرفع على الإنترنت + المزامنة بين كل الأجهزة

تحتاج خطوتين فقط: **(أ) إعداد Firebase** للمزامنة، ثم **(ب) رفع الملفات على GitHub Pages** للوصول من أي مكان. كل شيء **مجاني تماماً**.

---

### الخطوة (أ) — إعداد قاعدة بيانات Firebase

#### 1. إنشاء المشروع
1. افتح: <https://console.firebase.google.com>
2. سجّل الدخول بحسابك في Google
3. اضغط **"Add project"** أو **"إضافة مشروع"**
4. اختر اسماً مثل: `mazyoud-band` (أحرف إنجليزية)
5. **يمكنك تعطيل** Google Analytics (لسنا بحاجة لها)
6. انتظر إنشاء المشروع (~30 ثانية) ثم اضغط **Continue**

#### 2. تفعيل Realtime Database
1. من القائمة الجانبية اليسرى: **Build → Realtime Database**
2. اضغط **"Create Database"**
3. اختر منطقة قريبة (مثل `europe-west1` أو `asia-southeast1`)
4. اختر **"Start in test mode"** (مهم — سنغيّره لاحقاً)
5. اضغط **Enable**

#### 3. الحصول على الإعدادات
1. اضغط الترس ⚙️ بجوار "Project Overview" → **Project settings**
2. انزل لقسم **"Your apps"** → اضغط أيقونة الويب `</>`
3. اكتب اسماً مثل `MazyoudWebApp` ولا تختر "Firebase Hosting"
4. اضغط **"Register app"**
5. ستظهر لك قطعة كود تحتوي على `firebaseConfig` — **انسخها كاملة**

#### 4. لصق الإعدادات في البرنامج
افتح ملف `firebase-config.js` بمفكرة (Notepad) واستبدل القيم بالقيم التي نسختَها:

```javascript
window.FIREBASE_CONFIG = {
    apiKey: "AIzaSy...........",          // من firebaseConfig
    authDomain: "mazyoud-xxxx.firebaseapp.com",
    databaseURL: "https://mazyoud-xxxx-default-rtdb.firebaseio.com",
    projectId: "mazyoud-xxxx",
    storageBucket: "mazyoud-xxxx.appspot.com",
    messagingSenderId: "1234567890",
    appId: "1:1234567890:web:abcdef123456"
};
```

> 💡 **مهم**: تأكد من نسخ `databaseURL` كاملاً (يبدأ بـ `https://` وينتهي بـ `.firebaseio.com`).

#### 5. ضبط قواعد الأمان
1. عُد إلى **Realtime Database** → تبويب **Rules**
2. غيّر القواعد لما يلي:

```json
{
  "rules": {
    ".read": true,
    ".write": true
  }
}
```

3. اضغط **Publish**.

> 🔒 **ملاحظة أمنية**: هذه القواعد تجعل البيانات قابلة للقراءة/الكتابة من أي شخص يعرف رابط Firebase الخاص بك. للاستخدام الخاص داخل فرقتك المعتمدة هذا مقبول (رابط Firebase ليس عاماً افتراضياً). للحماية الأقوى، راجع قسم "الأمان المتقدم" في نهاية هذا الملف.

---

### الخطوة (ب) — الرفع على GitHub Pages

#### 1. إنشاء حساب GitHub
1. افتح <https://github.com> وأنشئ حساباً مجانياً (إن لم يكن لديك)

#### 2. إنشاء مستودع جديد (Repository)
1. اضغط **"New"** أو الزر `+` ثم **New repository**
2. اسم المستودع: `mazyoud-band` (أو أي اسم تريده)
3. اختر **Public** (مطلوب للنسخة المجانية من GitHub Pages)
4. **لا تختر** أي خيارات إضافية
5. اضغط **Create repository**

#### 3. رفع الملفات
بأبسط طريقة (بدون أوامر):

1. في صفحة المستودع الجديد، اضغط **"uploading an existing file"**
2. اسحب وأفلت كل هذه الملفات:
   - `index.html`
   - `app.js`
   - `style.css`
   - `firebase-config.js` (بعد تعديله بإعداداتك)
   - `README.md`
3. اضغط **"Commit changes"**

#### 4. تفعيل GitHub Pages
1. في صفحة المستودع → **Settings** (أعلى الصفحة)
2. من القائمة الجانبية اليسرى → **Pages**
3. تحت **"Source"** اختر **Deploy from a branch**
4. تحت **Branch** اختر **`main`** و `/(root)`
5. اضغط **Save**
6. انتظر دقيقة أو اثنتين، ثم حدّث الصفحة. ستظهر:

```
✅ Your site is live at https://USERNAME.github.io/mazyoud-band/
```

🎉 **افتح هذا الرابط من أي جهاز في العالم** — سيعمل البرنامج وتتزامن البيانات تلقائياً!

## 🧪 كيف تتأكد أن كل شيء يعمل؟

1. افتح الرابط على لابتوبك → سجّل دخول `admin` / `admin`
2. أضف عضواً تجريبياً وأنشئ له حساب دخول
3. افتح **نفس الرابط** على جوالك، وسجّل دخول بحساب العضو
4. على لابتوبك أنشئ مناسبة جديدة وحدد العضو كمشارك
5. **خلال ثانيتين**، شارة الإشعارات على جوال العضو سترتفع 🔔
6. اضغط قبول أو اعتذار من الجوال → فوراً يظهر التحديث على لابتوب المدير

في زاوية الشاشة السفلية اليسرى ستلاحظ شارة:
- ☁️ **متصل بالسحابة** = كل شيء تمام
- 📁 **يعمل محلياً** = Firebase لم يُعدّ بعد
- ⚠️ **خطأ** = راجع إعدادات `firebase-config.js`

## 🔐 الأمان المتقدم (اختياري)

للأمان الأعلى، يمكن استخدام Firebase Authentication بدلاً من القواعد المفتوحة:

```json
{
  "rules": {
    ".read": "auth != null",
    ".write": "auth != null"
  }
}
```

ثم ربط الدخول بـ Email/Password عبر Firebase Auth. هذا يحتاج تعديلات إضافية على الكود — راسلني إن أردتَ المساعدة.

## 🛠️ استكشاف المشاكل

| المشكلة | الحل |
|---|---|
| الشارة تظهر "يعمل محلياً" دائماً | راجع `firebase-config.js` — هل عدّلتَ `apiKey` و `databaseURL`؟ |
| الشارة تظهر "خطأ" أو "انقطع الاتصال" | افتح Console المتصفح (F12) لرؤية الخطأ التفصيلي. غالباً قواعد Realtime Database ليست `.read: true, .write: true`. |
| البيانات لا تظهر على جهاز ثاني | تأكد أن **كلا الجهازين** يفتحان نفس رابط GitHub Pages بنفس ملف `firebase-config.js`. |
| العضو لا يرى الإشعار فوراً | تحديث الصفحة قد يساعد. التحديث الدوري يحدث كل 60 ثانية، لكن المزامنة الفورية عبر Firebase تكون خلال ثانيتين عادةً. |
| ظهور رسالة خطأ JavaScript عند التحميل | تأكد أنك رفعتَ كل الملفات الخمسة معاً، خاصة `firebase-config.js`. |
